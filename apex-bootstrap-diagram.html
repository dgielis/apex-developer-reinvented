APEX Bootstrap

Oracle APEX Development Platform — One Command Setup
Infrastructure-as-Code for Oracle APEX: Database, ORDS, CI/CD, and Development Tooling
Command Modes

Five operational modes covering the full lifecycle from setup to teardown.

apex-bootstrap init
Full Platform Init
Complete greenfield setup: Docker containers, Oracle DB, ORDS, GitLab, Jenkins, scaffold project, wire CI/CD. Everything from zero to running.
apex-bootstrap add
Add Project
Adds a new APEX project to already-running infrastructure. Auto-detects running containers, discovers next available APP_ID, creates new schema & CI/CD jobs.
apex-bootstrap uninstall
Full Uninstall
Removes all bootstrap-owned Docker resources, cleans up installed packages (Homebrew/apt), deletes tracked repos and log files.
apex-bootstrap docker-clean
Docker Reset
Resets all Docker state — containers, volumes, networks, and cache — while preserving images. Nuclear reset for a fresh start.
apex-bootstrap vk
Vibe Kanban
Installs VK workstation toolchain: fnm, Node.js LTS, pnpm, Rust, cargo-watch, sqlx-cli, Claude Code, and clones vibe-kanban repo.
Init Flow — Step by Step

What happens when you run apex-bootstrap init -p "My APEX App"

🔍 Prerequisites Check
Scans system for all required tools and auto-installs anything missing.
  • Checks for: docker, docker-compose, git, curl, jq, rsync, sql (SQLcl), unzip, wget, perl
  • macOS → installs via Homebrew (installs Homebrew if needed)
  • Linux → installs via apt-get with sudo
  • Installs OpenJDK 21 for SQLcl and Jenkins
  • Starts Docker Desktop if not running (macOS)
bootstrap-cli.ts → checkPrerequisites()
1
📦 Extract & Template Assets
Embedded scaffold assets (base64 in the binary) are extracted and templated.
  • 15MB+ of base64-encoded scaffold files decoded
  • Template variables replaced: __PROJECT_NAME__, __PROJECT_SCHEMA__, __APP_ID__, etc.
  • Text files get variable substitution; binary files get direct decode
  • File permissions preserved from originals
scaffold-assets.generated.ts → extractAssets()
2
🐳 Docker Provisioning
Starts all infrastructure containers via Docker Compose with health checks.
  • Oracle Database 23c Free containers (DEV, TEST, optionally PROD)
  • ORDS 25.4.0 containers per environment
  • GitLab CE container for source control
  • Jenkins LTS container for CI/CD
  • Port conflict detection → auto-finds next available ports
  • Health-check polling until all databases report ready
docker.ts → dockerComposeUp() + waitForHealth()
3
🗄️ Database Bootstrap
Provisions schemas, workspaces, and connections inside each Oracle container.
  • Creates application schema & tablespace via SQLcl
  • Creates APEX workspace linked to schema
  • Configures ORDS for schema REST access
  • Saves SQLcl named connections (~/.sqlcl/connections/)
  • Sets up SYS and APP connection strings
provision.ts + bootstrap-env.ts → SQL scripts in database/deploy/
4
🚀 Application Deployment
Installs the APEX application, database objects, and REST modules.
  • Runs @install_source.sql for all DB objects (tables, views, packages)
  • Imports APEX application export (f1000/install.sql)
  • Installs ORDS REST modules (api.sql)
  • Deploys to DEV first, then replicates to TEST/PROD
deploy.ts → runSqlcl("@install_source.sql")
5
📂 Project Seed & Git Init
Initializes the project repo from templates with full variable substitution.
  • Exports SQLcl project metadata
  • Rsyncs vendor directories into repo structure
  • Template substitution across all text files in starter-template/
  • Git init → initial commit → push to local GitLab
  • Branch protection rules applied (develop, test, main)
seed-repo.ts → git init + gitlab.ts → createProject()
6
⚙️ CI/CD Wiring
Creates Jenkins jobs and GitLab webhooks for automated pipelines.
  • Creates Jenkins jobs: feature/*, develop, test, main branches
  • Configures GitLab → Jenkins webhooks for push events
  • Pipeline stages: checkout → deploy DB → import APEX → run tests
  • Approval gate for production (main branch) deployments
  • Writes local-bootstrap.json with full infrastructure report
delivery.ts + jenkins.ts + gitlab.ts
7
🛠️ Dev Tools & Skills
Installs companion skills and configures Claude Code integration.
  • Clones oracle-db-skills for database development assistance
  • Clones apex-component-modifier for APEX page editing
  • Configures MCP server for SQLcl ↔ Claude Code bridge
  • Displays final URLs: APEX app, ORDS, GitLab, Jenkins
bootstrap-cli.ts → installSkills()
8

Source Module Architecture

TypeScript modules compiled to a single Bun executable. Each module owns a specific concern.

bootstrap-cli.ts
CLI Entry Point
Command router, argument parsing, prerequisites checking, asset extraction orchestrator. The main 84KB file that ties everything together.
Core
init.ts
Init Orchestrator
Runs the full "init" pipeline: provision → deploy → seed-repo → delivery. Coordinates all other modules in sequence.
Core
add.ts
Add Orchestrator
Discovers running infrastructure from Docker, finds next available APP_ID, and adds a new project to existing containers.
Core
docker.ts
Docker Manager
Docker Compose orchestration, health-check polling, port allocation and conflict detection, container lifecycle management.
Infrastructure
provision.ts
DB Provisioner
Database environment setup: schema creation, APEX workspace setup, ORDS configuration, tablespace management.
Database
bootstrap-env.ts
Workspace Bootstrap
APEX workspace and application initialization via SQL. Creates the workspace, assigns schemas, sets up APEX app identity.
Database
deploy.ts
App Deployer
Initial APEX application deployment: runs install scripts for DB objects, imports APEX app export, installs ORDS modules.
Database
gitlab.ts
GitLab API Client
Full GitLab REST API integration: project creation, webhook management, access tokens, branch protection, clone URLs.
CI/CD
jenkins.ts
Jenkins API Client
Jenkins REST API: job creation from XML templates, build triggering, build status monitoring, approval gate configuration.
CI/CD
delivery.ts
CI/CD Orchestrator
Wires GitLab and Jenkins together: creates webhooks, configures pipeline jobs for all branch patterns, writes bootstrap report.
CI/CD
seed-repo.ts
Repo Seeder
Template extraction, variable substitution across all files, vendor directory sync, git init, initial commit and push.
Core
project-config.ts
Config Deriver
Derives all identifiers from project name: slugs, schema names, connection strings, ports, container names, ORDS paths.
Utility
sql.ts
SQLcl Runner
SQLcl subprocess management, command construction for both host and Docker execution, output parsing and error handling.
Database
connections.ts
Connection Resolver
Resolves SQLcl connection strings for all environments, handles host vs. Docker networking (localhost vs. host.docker.internal).
Database
progress.ts
Progress UI
Terminal progress tracker with ANSI colors, step/sub-step tracking, timing, and formatted status output.
Utility
bootstrap-state.ts
State Tracker
Persistent JSON state at ~/.apex-bootstrap/state.json. Tracks installed tools, repos, and logs for clean uninstallation.
Utility

Key Features
🎯
Single Binary Distribution
Compiled with Bun into one executable with all scaffold assets embedded as base64. Hand it to a colleague — no dependencies, no setup steps.
🌍
Multi-Environment
DEV, TEST, and optional PROD environments with isolated Docker networks, separate database containers, and independent ORDS instances.
🔄
Multi-Project Support
Run add to spin up additional APEX apps on the same infrastructure. Auto-discovers containers and allocates unique APP_IDs.
🖥️
Cross-Platform
Works on macOS (Homebrew) and Linux (apt-get). Detects platform, handles sudo, manages Docker Desktop vs. Docker Engine differences.
🧪
Testing Built-In
utPLSQL for PL/SQL unit tests and Playwright for browser-based APEX smoke tests. Both wired into CI pipeline from day one.
🤖
Claude Code Integration
Installs oracle-db-skills and apex-component-modifier skills, configures MCP server for SQLcl, enabling AI-assisted Oracle development.
🧹
Clean Uninstall
State tracking ensures uninstall only removes what bootstrap installed. Your pre-existing tools and Docker images stay untouched.
🔐
Port Conflict Resolution
Automatically detects port conflicts and finds the next available port via TCP bind testing. No more manual port juggling.
